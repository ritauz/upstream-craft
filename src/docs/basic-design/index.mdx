# 基本設計工程の全体像

基本設計工程は、要件定義で明確化された内容をもとに、**システムの構造・方式・振る舞いを設計する工程**です。  
この成果物は、詳細設計や実装の基盤となり、プロジェクト全体の品質と開発効率に直結します。

---

## 🚀 進め方の流れ

1. **システム方式設計**  
   - 構成方式、冗長化、ネットワーク、外部インターフェース方式を定義する。  

2. **機能設計**  
   - 画面、帳票、入出力データ、処理ルールを設計する。  

3. **データ設計**  
   - 論理データモデルを定義し、テーブル定義やデータ量見積りを行う。  

4. **非機能要件設計**  
   - 可用性・性能・運用・セキュリティ・移行・DR（災害対策）対応を設計に落とし込む。  

---

## 📑 プロセスと成果物の対応

| プロセス（やること） | 成果物（アウトプット） |
| --- | --- |
| システム方式設計 | **システム方式設計書**（構成図、ネットワーク構成図、外部IF一覧） |
| 機能設計 | **画面設計書**（UIプロトタイプ、画面遷移図）<br/>**帳票設計書**<br/>**入出力定義書** |
| データ設計 | **データベース論理設計書**（ER図、テーブル定義書、キー制約）<br/>**データボリューム見積もり表** |
| 非機能要件設計 | **非機能設計書**（クラスタ／フェイルオーバー設計、性能設計試算、監視設計、バックアップ設計、セキュリティ設計、移行計画、DR設計） |
