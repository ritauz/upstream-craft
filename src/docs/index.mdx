# 📘 プロジェクト標準化ドキュメント

このドキュメントは、**要件定義・基本設計フェーズにおけるプロセスと成果物**を体系的にまとめたものです。  
フレームワークやライブラリのドキュメントと同じく、**参照しながら実務に適用できる実践的なガイド**を目指しています。

---

## 🎯 このドキュメントの位置づけ
- 本プロジェクトにおける **標準的な上流工程の手引き**
- 成果物テンプレートと運用方法を **一元的に管理・参照できる場所**
- 経験が浅いメンバーでも、必要な流れとアウトプットを **すぐに理解できる仕組み**

---

## 👥 対象読者
- **プロジェクトマネージャー（PM）**  
  提案～計画段階で成果物を抜け漏れなく揃えたい方
- **システムエンジニア（SE）**  
  要件定義・基本設計を効率よく進めたい方
- **レビュー担当者 / 上長**  
  成果物の完成度や進め方を確認したい方

---

## 📚 どのようなときに読むか
- 新しい案件で **要件定義や基本設計を開始するとき**
- 成果物テンプレートの **使い方を確認したいとき**
- 他メンバーと **認識を揃えるための共通言語** が欲しいとき
- レビューの観点や、作業の抜け漏れチェックを行いたいとき

---

## 🚀 このドキュメントでできること
- **プロセスごとの流れ**（やること → 成果物）を理解できる
- **テンプレートをダウンロード**してすぐに使える
- **依存関係**や参照すべきドキュメントを把握できる
- プロジェクトを通して **共通の標準**を持ち、属人化を防ぐ
