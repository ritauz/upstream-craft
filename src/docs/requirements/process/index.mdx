# 要件定義プロセス解説

このセクションでは、要件定義工程を進める際の **具体的な進め方・手順・注意点** をまとめています。  
顧客と開発チームの間で「何を」「どのように」合意形成するかを体系的に示します。

---

## 📌 この階層の位置づけ
- **目的**: 業務要件・機能要件・非機能要件を明確化し、成果物（一覧書や定義書）につなげる  
- **内容**: 各プロセスごとの進め方・留意点・チェックリスト

---

## 🗂️ ここに含まれるドキュメント
- 業務要件の明確化
- 機能要件の整理
- ユースケース図・業務フロー図の作成
- 非機能要件の合意形成
- 制約条件・適用範囲の確認
