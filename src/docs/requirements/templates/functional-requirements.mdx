# 機能要件一覧

## 🎯 目的
業務要件を実現するために必要なシステム機能を整理し、顧客と合意するためのテンプレート。  
業務要件との対応を明示することで、**抜け漏れ防止**と**設計へのスムーズな接続**を支援する。

---

## ✍️ 埋め方のポイント
- **利用可否**：この機能を対象とするかどうかを明記  
- **ID**：一意の番号を付与（例：FR-001）  
- **業務ID**：対応する業務要件IDを記載（例：BR-001）  
- **機能カテゴリ**：検索／登録／承認／通知など大分類を設定  
- **機能名**：ユーザーに伝わる自然言語で短く記載（例：「在庫検索」）  
- **説明**：「誰が／何を／なぜ」を1〜2文で要約（例：「倉庫担当が在庫を検索し、欠品確認を行うため」）  
- **重要度**：高／中／低で業務への影響度を評価  
- **状態**：ドラフト／レビュー中／承認済／却下で管理  
- **備考**：特記事項や補足を記載  

---

## 👀 マネジメント観点での使い方
- **業務要件と機能要件のトレーサビリティ**を確保することで、顧客要件がシステム機能に正しく反映されているかを確認可能  
- 「重要度」を付与することで、開発リソース配分や優先度判断に活用できる  
- レビュー観点：
  - 機能が業務要件に対応しているか  
  - 説明がユーザー目線になっているか  
  - 過剰機能や顧客価値が不明確な機能が含まれていないか
